Security Testing on Zigbee Protocol Implementations

Research project to apply state-of-art software testing techniques on Zigbee protocol

With the increasing popularity of the Internet of Things (IoT) devices, Zigbee protocol has become a dominant wireless protocol to reduce the power and memory cost of IoT devices. It has been widely adopted in home automation, green power, and manufacturing. Recently, server vulnerabilities in Zigbee protocol implementations have compromised IoT dvices from different manufactuers. It becomes imperative to perform security testing on Zigbee protocol implementations. Thus, this research project aims to apply existing state-of-art vulnerability detection techniques, such as fuzzing and symbolic execution, to Zigbee protocol implementations

Challenges

It's not a trivial task to directly apply the existing vulnerability detection techniques, such as fuzzing and symbolic execution to Zigbee protocol implementation. In particular, it remains a significant obstacle to deal with vendor-specific configuration and low-level hardware events.
  • Vendor-specific Toolchain/Configuration The Zigbee protocol vendor could specify a particular development toolchain or programming configuration for their protocol implementations, which prevents user from applying general but unsupported application on their implementations.
    • Compiler Check: prevent general compiler like GCC and LLVM from compiling source code
    • Vendor specified programming keywords: unable to parse with existing code analysis application
  • Close Dependency on Hardware Configuration Instead of transmitting data through Internet, the Zigbee protocol transmits data via radio channel; thus, it has close dependencies on the embedded device's hardware configuration. It's a big challenge to simulate Zigbee protocol execution in a software environment.
    • Not compatible with simulator requiring Linux kernel or an abstraction layer.
    • Vendor-specific emebedded devices are not supported by most existing simulators.
    • Require significant engineering efforts to support all device-specific peripherals and new embedded chips.

Ongoing Work

Though Z-Fuzzer has detected several security vulnerabilities on Zigbee protocol implementations, its performance is not efficient since it fails to test exercise execution paths that requires satisfying particular condition examination. Currently, we are working on a hybrid testing solution on Zigbee protocol, which combines fuzzing and symbolic execution, to detect security vulnerability. The main idea is to apply symbolic execution on Zigbee source code to record symbolic states, and cooperate fuzzing with symbolic execution when fuzzing rarely exercises new execution paths.
